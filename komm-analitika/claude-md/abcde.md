## Что решает этот дашборд
Оценивает качество трафика и честно сравнивает менеджеров: где горячий трафик сгорает у слабого продавца, а где слабый трафик вытягивают.

## Шаг 1 · Проверь пререквизиты (что должно быть готово в инфраструктуре)
Спроси пользователя и убедись, что всё это есть в его CRM/данных. Если чего-то нет — честно скажи, что без этого дашборд соберётся неполным, и предложи минимальный рабочий вариант.
- У каждого лида в CRM проставлен сегмент A / B / C / D / E (командой; в идеале ИИ-контроль качества слушает звонок и проверяет корректность сегмента)
- Квалификация ставится после первого разговора

## Шаг 2 · Метрики и формулы
- **Доли A / B / C / D / E** по дням.
- **КВАЛ-1** = (A + B + C) / все лиды.
- **КВАЛ-2** = (A + B) / все лиды.
- **Доля D + E**.
- **Конверсия по каждому сегменту у каждого менеджера** против бенчмарков:
  - A: 60–80%
  - B: 20–40%
  - C: 5–15%
  Это ловит аномалии — например, «горячий трафик (A) сгорает у слабого менеджера».

## Шаг 3 · Фильтры (срезы)
- Период
- Канал
- Менеджер
- Продукт

## Шаг 4 · Вёрстка (сверху вниз)
Собери в порядке: шапка с фильтрами → цель-метрики (план-факт, светофор 🟢🟡🔴 где задан план) → графики динамики по неделям → детализация (drill-down: список конкретных сделок/клиентов с ID-ссылкой в CRM, сортировка по проблемному показателю).

Особое для этого дашборда — красные зоны:
- КВАЛ-1 < 35% (или две недели подряд < 50%).
- Доля D + E > 45%.
Подсвечивай их 🔴 и выводи наверх.
