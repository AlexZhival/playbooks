🧠 Второй мозг — как развернуть

Второй мозг — не отдельное приложение и не самописный бот, а папка, которую открывает Claude Code, плюс Telegram как интерфейс и коннекторы к вашим данным только на чтение. Всё работает на вашей машине, наружу ничего не выгружается. Отдельный API-ключ Anthropic не нужен — работает на вашей подписке Claude.

⬇️ ссылка на стартовый набор «Второй мозг»

Порядок: откройте папку набора в VS Code → запустите Claude Code → вставьте мастер-промт (раздел 2) — Claude Code проведёт вас за руку. Делайте после собранной обёртки/контекста: ум мозга = качество контекста, который вы в него положили.

0 Как это устроено (30 секунд)

ЧастьЧто это
🧠 Мозгпапка + CLAUDE.md (главная инструкция) + context/ (ваши опорные материалы)
💬 ИнтерфейсTelegram через официальный плагин Claude Code — пишете/наговариваете, отвечает Claude Code на вашем компьютере
🔌 Данныеread-only коннекторы Claude (Drive / Calendar / Notion) + read-only скрипты (предиктивная модель, CRM, дашборды)
🔒 Безопасность.claude/settings.json (что можно / что запрещено) + правила в CLAUDE.md

1 Предпосылки

  • VS Code + Claude Code (вход выполнен, активная подписка Claude)
  • Отдельный дополнительный Telegram-аккаунт под бота (не основной рабочий и не личный) с 2FA
  • Токен бота от @BotFather (/newbot → имя + username на bot → токен)

2 Мастер-промт (вставьте в Claude Code)

Прочитай файл START-HERE.md целиком и веди меня по настройке второго мозга
строго по шагам и строго по разделу «Безопасность».
Правила:
— Не выполняй ни одного шага без моего подтверждения.
— Токен бота я настрою сам через /telegram:configure, в чат его не вставляю.
— Собираем ядро: агент отвечает на вопросы по материалам этой папки,
  доступ в Telegram — только моему аккаунту (allowlist).
— Ничего не удаляй, наружу ничего не пиши. Все интеграции — только на чтение.
Порядок:
1) проверь предпосылки и .claude/settings.json;
2) помоги собрать архитектуру контекста — создай папку context/ со структурой
   (компания, аудитория/JTBD, позиционирование, tone of voice, продукты и цены,
   клиенты, конкуренты), задай мне вопросы и помоги заполнить CLAUDE.md,
   я вставлю свои опорные материалы;
3) настрой Telegram-плагин.

3 Соберите архитектуру контекста (это и есть «мозг»)

Шаг 3.1 — создайте папку проекта (например vtoroy-mozg/) на диске или в облаке и сложите в неё все материалы обёртки, которые вы собирали ранее: JTBD (jobs to be done), tone of voice, бриф о компании, позиционирование/отстройка, продукты и цены, клиенты, конкуренты, презентации. Это «пища» мозга — чем полнее контекст, тем умнее ответы.

Шаг 3.2 — CLAUDE.md — главная инструкция: кто вы, что за компания, целевая аудитория (ICP/JTBD), продукты, цены, tone of voice, как отвечать. В наборе есть шаблон — Claude Code задаст вопросы и поможет заполнить.

Шаг 3.3 — папка context/ — разложите опорные материалы по файлам (скелет создаст агент, вы наполните из уже собранных материалов):

context/
├── компания.md          # о компании, видение, миссия
├── аудитория-jtbd.md    # ICP, сегменты, Jobs To Be Done
├── позиционирование.md  # отстройка от конкурентов
├── tone-of-voice.md
├── продукты-и-цены.md
├── клиенты.md
└── конкуренты.md

Викилинки [[имя]] (по желанию): внутри файлов ссылайтесь друг на друга — [[продукты-и-цены]]. В CLAUDE.md задано правило открывать такие ссылки. Полезно, когда база растёт.

💡 PDF-материалы перед загрузкой прогоните через markitdown (бесплатный инструмент Microsoft) — он переведёт их в текст, и мозг не будет жечь токены при каждом чтении.

4 Подключение Telegram (официальный плагин)

  1. /plugin install telegram@claude-plugins-official
  2. /reload-plugins
  3. /telegram:configure <ваш_токен> — токен уходит в защищённый файл плагина, не в проект и не в git
  4. Закройте текущую сессию Claude Code, откройте терминал в папке проекта и запустите:
    claude --channels plugin:telegram@claude-plugins-official
    Держите терминал открытым — пока он жив, бот работает.
  5. Напишите боту в личку → 6-значный код → /telegram:access pair <код>
  6. Замок: /telegram:access policy allowlist
🔒 Allowlist не пропускайте. Без него любой, кто узнает username бота, сможет командовать вашим проектом и читать файлы.
💡 Как поднять заново: зашли в папку проекта → claude --channels plugin:telegram@claude-plugins-official → бот на связи.

5 🔒 Безопасность (обязательно)

  • 2FA Telegram: Settings → Privacy and Security → Two-Step Verification. Завершите лишние Active Sessions.
  • Только Bot API @BotFather. Никогда: userbot (вход по номеру), QR-подключение личного Telegram в сторонние сервисы. 🚩 Просят «войти по номеру / QR / полный доступ» — СТОП.
  • Замок .claude/settings.json (уже в наборе): deny — rm -rf, sudo rm, curl … | bash, чтение секретов (.env, *.key, *.token); allow — только Read/Grep/Glob, WebSearch/WebFetch, читающие методы коннекторов и ваши скрипты по имени.
  • Отдельный доп. аккаунт · токен через плагин, не в чат · allowlist только ваш ID · всё только на чтение · секреты — в отдельные файлы вне git.
  • Kill switch: @BotFather /revoke → /telegram:access policy disabled → при желании /deletebot.

6 Коннекторы данных (Фаза 2, по желанию, только чтение)

claude.ai → + → Connectors → Google Drive, Google Calendar, Notion. Разрешены только читающие методы. Годятся для простых доков и разовых вопросов.

7 Живая предиктивная модель через бота

Ваша модель — Google Sheets с несколькими вкладками. Коннектор Drive читает обычно только первый лист и не работает по расписанию. Два варианта — от простого к надёжному:

Вариант A · просто (5 минут)

Сделайте в таблице первую вкладку «Сводка» и вынесите на неё ключевые цифры формулами с других вкладок (прогноз, план-факт, run-rate, светофор). Коннектор Drive читает первый лист — бот отвечает по этой сводке по ссылке. Минус: только то, что на «Сводке»; по расписанию не шлёт.

Вариант B · надёжно (Google Sheets API, read-only)

  1. В Google Cloud Console создайте сервисный аккаунт, включите Google Sheets API, скачайте JSON-ключ (в файл вне git).
  2. Расшарьте таблицу на email сервисного аккаунта с правами Viewer.
  3. Claude Code напишет скрипт: spreadsheets.values.get(spreadsheetId, 'Вкладка!A1:Z100'), scope spreadsheets.readonly.
  4. Скрипт — в белый список; бот вызывает его по запросу «дай статус по модели» и по расписанию.

readonly — изменить/удалить в таблице невозможно. Начните с A; нужна автоматика и любые вкладки — переходите на B.

8 Проверка

Сначала — быстрый тест связи: «Прочитай CLAUDE.md и расскажи, какие правила там описаны». Ответил по сути файла — связь есть. Дальше по данным: «Какой прогноз выручки на месяц и где красная зона?» → мозг прочитает контекст/модель и ответит. ✅ Готово. Готовые промты — в ПРОМТЫ.md набора.

9 CRM и дашборды (продвинутое, по желанию, строго read-only)

Подключение к CRM для сбора данных: заведите отдельного интеграционного пользователя / API-ключ с ролью «только просмотр» · токен вне git, чтение агенту запрещено · один GET-скрипт в белом списке · сначала тестируйте на копии. Без прав на запись — удалить или задублировать базу невозможно. CRM без read-only роли — не подключаем напрямую, а экспортируем копией.

Внимание: бывали случаи, когда агент с правами на запись сносил или дублировал базу CRM. Подключайте только на чтение и с подтверждением действий — так база пользователей и данные не пострадают и не утекут.

🔑 Логин и пароль на страницы с дашбордами

Когда Claude Code соберёт вам дашборды как веб-страницы — обязательно закройте их простым логином и паролем (gate-скрипт есть в наборе). Иначе ссылка с коммерческими данными будет открыта любому, у кого она есть. Промт: «добавь на страницу дашборда вход по логину и паролю, пароль храни вне кода».

10 Ритуалы и кроны

Настройте повторяющиеся задачи на конкретный день/время: вечерний светофор-дайджест по отделам из предиктивной модели, run-rate алерт, сводка SLA/TLT + дожим, крон-ресёрч по нише, аудит недели. Голосом: «каждый будний день в 19:00 собери сводку по отделам и пришли в рабочую группу».

11 Команда и хостинг

  • Доступ = allowlist Telegram. Новый — добавили ID; уволили — убрали ID.
  • С любого устройства — бот доступен каждому из белого списка.
  • 24/7 и всем — вынесите мозг на всегда-включённую машину (Mac mini / сервер).
  • Знания — общий Google Drive; коннекторы, сервисный аккаунт и CRM — на служебном аккаунте компании.
  • Офбординг ключевого человека: убрать из allowlist и доступов Drive, перевыпустить токен бота, ключ сервисного аккаунта и CRM-ключ.

Чего НЕ делаем

  • ❌ userbot и QR-подключения личного Telegram
  • ❌ права на запись/удаление в CRM, модели, боевых системах
  • ❌ вставлять токены и ключи в чат
