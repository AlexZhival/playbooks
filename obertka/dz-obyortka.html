Домашнее задание — Обёртка компании + опорные материалы

Что делаем: собираем «обёртку» — стратегический слой смыслов о бизнесе (бриф, JTBD, сегменты, Value Proposition, Tone of Voice, видение). Складываем в единую папку-«мозг». Это фундамент второго мозга и всех следующих модулей (предиктивные модели → OKR → координации).

Дедлайн: до следующей встречи. Принцип: «5 раз подумай — 1 раз сделай»: сначала контекст и смысл, потом инструмент.

🔑 Нужна подписка Claude Pro или Max — для обоих вариантов (и Claude Code, и Проекты, и режим Research есть только на платном тарифе).

❓ Незнакомый термин (JSON, репозиторий, деплой, MCP…) — смотрите Глоссарий простыми словами в разделе 7.

1 · Сначала распределите работу в команде

❗ Не нужно, чтобы все бросались делать домашку. Обёртка собирается по брендам, а не каждым сотрудником. На бренд назначьте одного «сборщика» — он ведёт инструмент и собирает обёртку; остальные по ролям поставляют материалы и проверяют свою часть, а не запускают то же самое параллельно.
  • Одна папка = один бренд, а не один человек. Сначала фиксируем архитектуру брендов (зонтичный бренд, саб-бренды, продукты) — из неё вырастает структура папок.
  • Инструмент — «однопользовательский». И папка проекта в Claude Code (Вариант A), и Проект в claude.ai (Вариант B) — под одним человеком. Несколько людей одновременно в одной и той же обёртке не соберут — будет рассогласование. Поэтому:
    • на бренд — один сборщик, который ведёт инструмент от начала до конца;
    • нужно параллелить — параллельте по разным брендам (свой сборщик и своя папка на каждый), а не сажайте несколько человек на одну обёртку;
    • остальные роли скидывают сборщику материалы (сайт, цифры, транскрипты) и валидируют свой блок на разборе.
  • Кто что поставляет по ролям (кто ближе к данным): бриф и цифры/воронки — финансы + продажи; JTBD, сегменты, Value Proposition, Tone of Voice — маркетинг + продукт; конкуренты — маркетинг; видение и миссия — собственник / HR.
  • У кого материалы по компании уже собираются — используем их как есть, не пересобираем с нуля.

Заполните карту брендов (по ней нарежем папки и ответственных) и пришлите её в рабочий чат команды:

• Зонтичный бренд: …
• Саб-бренды: …
• Продукты / направления под каждым: …
• Кто ответственный за каждый бренд: …

2 · Как проходить — два варианта

Оба дают одинаковый по качеству результат смысловой обёртки. Разница — где вы работаете.

рекомендуем · полный

Вариант A · VS Code + Claude Code «за руку»

Собираете обёртку в связке VS Code + Claude Code: ИИ видит всю папку проекта, сам читает ваши файлы, отвечает на вопросы из них и сохраняет артефакты в папку. Здесь работают скиллы и markitdown.

🛠️ Установку и первичную настройку удобнее сделать с техническим специалистом (если в команде есть такой человек): поставить VS Code + расширение Claude Code, войти по подписке, выставить настройки. Дальше собираете обёртку сами.

📘 Установка + настройка⬇️ Стартовый набор
без установки · смысл

Вариант B · «Проекты» в Claude (браузер)

Ничего не ставите. Создаёте Проект на claude.ai, загружаете материалы, идёте по рабочей тетради — копируете промпты по одному, ответы сами сохраняете в свою папку.

Проекты на тарифе Pro/Max. Здесь нет установки скиллов и автосохранения — вместо Deep Research режим Research, файлы складываете руками.

📗 Конвейер по слоям📊 Рабочая тетрадь

Вариант A — по шагам (подробно)

Полные памятки с картинками: Установка · Как пользоваться.

  1. Поставьте VS Code. Откройте code.visualstudio.com → «Download» → перетащите приложение в «Программы» → запустите. VS Code — это ваше рабочее окно (редактор).
  2. Поставьте расширение Claude Code. Слева иконка Extensions (Cmd/Ctrl + Shift + X) → в поиске Claude Code → выберите расширение от Anthropic → Install. Если попросит — перезапустите VS Code.
  3. Войдите по подписке. Откройте панель Claude Code (значок ✦) → Sign in → аккаунт с Pro или Max.
  4. Выставьте настройки один раз (меню режимов, Shift + Tab): Mode → Edit automatically (сам сохраняет артефакты, опасное всё равно спросит) · Effort → High · Model → Opus (/model). ⚠️ Bypass permissions — НЕ включать. Контроль контекста: /context, /compact, /clear, расход — /usage.
  5. Скачайте и распакуйте стартовый набор. Внутри: CLAUDE.md (мозг-инструкция), конвейер/, материалы-компании/, обёртка/.
  6. Сложите ВЕСЬ существующий контекст компании в папку материалы-компании/ — чтобы, выполняя CLAUDE.md, Claude уже начал сам отвечать на вопросы брифа из этих файлов:
    • сайт — сохранить ключевые страницы целиком через расширение FireShot / GoFullPage (страница → PDF/картинка); презентации / питч-деки / КП, прайсы и тарифы, описания продуктов;
    • цифры: выручка, средний чек, конверсии, выгрузки воронок из CRM;
    • транскрипты встреч, интервью с основателем, отзывы дословно.
    • PDF, книги, YouTube-видео → сначала переведите в Markdown (файлы .md) сервисом markitdown: PDF беспощадно жрёт токены, а .md Claude читает чисто и дёшево. Проще всего прямо в Claude Code: «установи markitdown и переведи эти PDF и ссылки на YouTube в .md, положи в материалы-компании/». Утилита: github.com/microsoft/markitdown.
  7. Откройте папку и запустите сборку. В VS Code File → Open Folder → папка проекта. В панели Claude отправьте: 
    Прочти CLAUDE.md и начни вести меня по сборке обёртки
    Дальше отвечаете на его вопросы. Он читает материалы-компании/, отвечает по фактам, помечает 🔴 «нет данных» / 🟡 «частично», ведёт по конвейеру и сохраняет артефакты в обёртка/.

Вариант B — по шагам

  1. Создайте место для результатов. Папка <Бренд>/Обёртка/ на Google Диске или локально — сюда 6 файлов (01-бриф.md … 06-видение.md). Это ваша «папка-мозг».
  2. Создайте Проект. claude.ai слева → Projects → New project. Назовите по бренду; в описании — кто вы и чем занимаетесь.
  3. Загрузите ВСЕ материалы в проект (кнопка «+»): сайт (через Chrome-расширение GoFullPage — страница целиком в PDF), презентации, прайсы, цифры/воронки, транскрипты. Большие PDF/книги/YouTube лучше сначала в .md через markitdown (см. Вариант A / технический коллега).
  4. Откройте рабочую тетрадь. Вкладки снизу — под каждый артефакт (раздел 3). Общий цикл промптов — во вкладке 🚀 Промпт.
  5. Собираете артефакты по порядку (раздел 3). Для каждого — новый чат в Проекте, промпты вкладки сверху вниз, все в одном чате: ⚙️ Системный (метит 🔴/🟡) → 📋 Основной (вопросы блоками по 20–30, в брифе ~200) → ❓ Уточняющий (приоритеты 🔥/⚡/🟡) → ✅ Финальный (аудит).
  6. Сохраняете ответ в свой файл и переходите к следующему артефакту.
💡 Не знаете, как сформулировать промпт? Попросите Клод: «действуй как prompt-engineer и собери мастер-промпт под эту задачу» — он выдаст цепочку синхронизированных промптов до результата.

3 · Артефакты обёртки — что собираем по порядку

Идём строго по порядку — каждый следующий опирается на предыдущий. Где данных нет — 🔴, где частично — 🟡. Финальное слово всегда ваше. Метод каждого шага — Алгоритм AI-домашки.

4 · Чек-лист недели

  1. Забронировать слоты под домашку · заполнить и прислать карту брендов (раздел 1)
  2. A Поставить VS Code + Claude Code  ·  B Открыть claude.ai → создать Проект и папку <Бренд>/Обёртка/
  3. Собрать и загрузить материалы компании (сайт, презентации, прайсы, цифры/воронки, транскрипты; PDF/YouTube → в .md через markitdown)
  4. A Поставить скиллы (раздел 5)  ·  B для конкурентов включать режим Research
  5. Пройти артефакты по порядку (раздел 3), прогоняя промпты вкладки в одном чате
  6. Валидировать: закрыть все 🔴 и 🟡, сохранить 6 артефактов в папку обёртки
  7. Свериться с безопасностью (памятка): ключи не в git · потолок расходов · ИИ не трогает прод одной командой
  8. (по желанию / отдельный трек) — второй мозг в Telegram (раздел 9)

5 · Скиллы (Вариант A · Claude Code) и инструменты

Скилл = папка с инструкцией («профессия»), которую Claude сам подтягивает под задачу — не нужно каждый раз объяснять «как». Ставятся в Claude Code → Skills → Create New Skill (или через /plugin): создаёте папку скилла и кладёте туда файлы из репозитория. Аудит перед установкой обязателен — открываете репозиторий и смотрите, что внутри.

🟩 Claude прямо в Excel

Claude умеет работать внутри Excel боковой панелью: читает таблицу, считает, строит финмодели и объясняет формулы обычными словами. Доступно на планах Pro / Max / Team / Enterprise.

  • Что нужно: современный Excel с подпиской Microsoft 365 — на Windows, Mac, в браузере или на iPad. ⚠️ Excel 2016/2019 и личные @outlook.com-аккаунты не поддерживаются (нужен рабочий аккаунт M365).
  • Установка (≈1 минута): откройте Excel → Home → Add-ins (на Mac: Tools → Add-ins) → в поиске наберите Claude → Add / Activate. Либо через Microsoft AppSource — «Claude for Microsoft 365» → Get it now.
  • Вход: справа откроется панель Claude → Sign in вашим аккаунтом Claude (Pro/Max) → готово, спрашивайте прямо по таблице.

Официальная инструкция: support.claude.com — Use Claude for Excel.

7 · Глоссарий — простыми словами

Короткий словарь терминов, которые чаще всего встречаются в вайб-кодинге. Не нужно знать их все наизусть — возвращайтесь сюда, когда встретите незнакомое слово.

ТерминЧто это понятным языком
ПромптЗапрос, задание для ИИ — то, что вы пишете или надиктовываете обычными словами.
КонтекстВсе материалы и факты, которые ИИ «видит» и на которые опирается в ответе. Чем он полнее — тем лучше результат.
Токены«Кусочки текста», которыми ИИ измеряет объём. Больше текста → больше токенов → дороже и медленнее. Поэтому большие PDF переводим в .md.
Скилл (Skill)Папка-инструкция, «профессия» для Claude (маркетолог, дизайнер, аналитик). Claude сам подтягивает её под задачу.
СубагентОтдельный «помощник» Claude под конкретную подзадачу — работает параллельно, чтобы сделать больше и качественнее.
MCP«Переходник», через который Claude подключается к вашим сервисам: Почта, Диск, CRM, Календарь, Telegram.
Папка проектаОбычная папка на компьютере, которую открывает Claude Code и видит все файлы внутри. Это и есть ваш «мозг».
CLAUDE.mdФайл-«мозг» проекта: роль Claude, правила и порядок работы, по которому он ведёт вас за руку.
Markdown (.md)Простой текстовый формат с лёгким оформлением (заголовки, списки). ИИ читает его чисто и дёшево — наш основной формат заметок.
JSONТекстовый формат для данных вида «ключ: значение». Так программы обмениваются структурированной информацией между собой.
Репозиторий (repo)Папка с проектом или скиллами, обычно на GitHub. Её можно посмотреть и скачать к себе.
GitHubСайт-хранилище репозиториев — там лежат скиллы, шаблоны и код, которыми делятся разработчики.
GitСистема, которая хранит историю изменений файлов: кто, что и когда поменял. Позволяет откатиться назад.
.envФайл с секретами (ключи, пароли). Его никогда не выкладывают в общий доступ и в git.
APIСпособ одной программе обращаться к другой (например, к ИИ) по «протоколу». Часто — через API-ключ.
API-ключ«Пароль» для доступа к сервису по API. Хранится в секрете, как и любой пароль.
Терминал / командная строкаОкно, куда вводят текстовые команды. Claude Code часто работает через него — но команды набирает он, не вы.
Деплой (deploy)«Выкатить» готовый продукт в интернет или на сервер, чтобы им можно было пользоваться по ссылке.
ХостингСервис, где «живёт» ваш сайт или приложение после деплоя.
Локально / прод«Локально» — на вашем компьютере (тест, безопасно, 🟢). «Прод» (продакшн) — боевая версия для реальных пользователей (🔴 осторожно).
АртефактГотовый результат-файл, который собрал ИИ: бриф, лендинг, модель, дашборд.
ОбёрткаПапка со смыслами о вашем бизнесе (бриф, JTBD, сегменты, Tone of Voice, видение) — фундамент второго мозга. Именно её вы собираете в этой домашке.
Вайб-кодинг (vibe coding)Сборка рабочих инструментов «на словах»: вы описываете задачу — ИИ пишет код и собирает результат.
Викилинк [[…]]Ссылка между файлами внутри вашей папки-мозга. Связывает related-заметки, чтобы контекст был единым.

8 · Безопасность — 3 правила

  • Ключи и пароли — никогда в git. .env в .gitignore, боевые ключи — в хранилище хостинга.
  • Потолок на расходы у ИИ-сервиса и хостинга.
  • ИИ не трогает прод/базу одной командой — опасное только через ваше подтверждение.

Зоны: 🟢 локально/тест — смело · 🔴 прод/деньги/чужие данные — с оглядкой. Полная памятка: Безопасность.

9 · Второй мозг в Telegram — по желанию / отдельный трек

Когда обёртка собрана — на её основе поднимается «второй мозг»: бот в Telegram, который держит весь контекст, читает данные (только на чтение), приносит алерты и разборы. Это не отдельное приложение, а папка-проект, которую открывает Claude Code, плюс Telegram как интерфейс.
