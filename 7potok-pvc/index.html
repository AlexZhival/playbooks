Собери свой первый рабочий инструмент — из своих же опорников

Вы уже собрали смыслы: бриф, JTBD, Tone of Voice, позиционирование, предиктивные модели. Вы видели живую сборку модели SoftSnow за 30 минут. Теперь — то же самое, но руками. Берёшь одно бутылочное горлышко своего отдела и за один вечер превращаешь его в инструмент, который реально работает.

00

Что ты делаешь в этом ДЗ

Не распыляйся. Не делай 33 агента сразу. Один фокус — самое больное измеримое горлышко, которое сильнее всего блокирует следующий уровень. Починка именно его даёт максимальный импакт.

Шаг 1
Выбери одно бутылочное горлышко (метрика, за которую отвечаешь)
Шаг 2
Поставь окружение: VS Code + Claude Code + голос
Шаг 3
Собери инструмент по алгоритму 7 шагов из своих опорников
Шаг 4
Оживи его: дашборд / отчёт / алерт в Telegram
Критерий сдачи (Definition of Done): инструмент открывается в браузере / приходит в чат и реально работает на твоих цифрах. Не «красивая табличка», а штука, которой ты пользуешься в понедельник.
01

Рабочее место за 6 шагов

Терминал не нужен. Ставим VS Code, внутрь — расширение Claude Code, входим по подписке, добавляем голос. Делай сверху вниз. Если ты в России — читай примечание про авторизацию в конце раздела.

1
Поставить VS Code
  1. Открой code.visualstudio.com → Download (macOS или Windows).
  2. Перетащи приложение в «Программы» (Mac) или установи (Windows) и запусти.
Готово, когда: VS Code открылся.
2
Поставить расширение Claude Code
  1. Слева — иконка Extensions (кубики) или Cmd ⇧ X / Ctrl ⇧ X.
  2. В поиске набери Claude Code → выбери расширение от Anthropic → Install.
  3. Если попросит — перезапусти окно: Cmd ⇧ P → Reload Window.
Готово, когда: у Claude Code появилась кнопка Uninstall / Disable.
3
Войти по подписке (Pro / Max)
  1. Открой панель Claude Code (значок ✱ справа вверху) → Sign in.
  2. Войди в аккаунт Claude, где активна Pro или Max. Это не API-ключ — просто подписка.
Готово, когда: в панели появилось поле для ввода команды.
4
Выставить настройки (один раз)

Меню режимов: Shift Tab или из панели. Подробности — в блоке ниже.

Коротко: модель Opus, уровень размышлений повыше середины, режим Edit automatically (не Bypass).
5
Подключить голосовой ввод 🎙️

Александр диктует почти всё — клавиатура лежит без дела. Это не понты, а скорость: думать вслух быстрее, чем печатать, а контекст получается богаче. Поставь приложение голосового ввода, которое печатает текст в любое поле (в т.ч. в панель Claude Code).

  • Wispr Flow — кроссплатформенно (Mac / Windows), диктовка в любое окно. wisprflow.ai
  • superwhisper — для Mac, на базе Whisper, работает офлайн. superwhisper.com
Готово, когда: зажал хоткей, сказал — текст появился в панели Claude Code.
6
Поставить лимит бюджета (безопасность)

Это первое, что делаем по безопасности. Чтобы оставленный «думать на ночь» агент не списал лишнего и чтобы видеть аномалию (вдруг хакнули).

  1. Зайди в Anthropic Console → Settings → Limits / Billing: console.anthropic.com
  2. Поставь дневной/месячный лимит (ориентир Александра — ~100–150 $/день как «сигнальный потолок»).
Реальность: на Max за месяц редко выходит дороже ~200 $. Но потолок ставим сразу.

⚙️ Рекомендуемые настройки

Для тех, кто собирает в первый раз. Меняется в один клик в панели Claude Code.
МОДЕЛЬ
Opus — самая гибкая, тянет глубокие задачи. Переключение: /model
РАЗМЫШЛЕНИЯ
Ползунок «effort»: чем выше — тем глубже думает и больше тратит токенов. Для большинства задач — чуть выше середины; для важной аналитики — предпоследний.
РЕЖИМ
Edit automatically — сам сохраняет файлы в папку, опасное всё равно спросит. «Bypass permissions» — НЕ включать, особенно если рядом боевая база.
КОНТЕКСТ
Окно забилось — попроси «дай промпт для нового окна, чтобы продолжить с того же места», открой новое и вставь. Память подтянется из папки. Так работаешь безлимитно.
🇷🇺 Если ты в России: не прыгай постоянно между странами через дешёвые VPN и не дёргай авторизацию — Anthropic считывает это как подозрительное поведение и может заблокировать аккаунт. Перед регистрацией почитай актуальную инструкцию по безопасной авторизации. Прецеденты блокировок уже есть.
02

Папка проекта и репозиторий

Вся суть вайб-кодинга — одна папка проекта, в которую ты складываешь контекст. Внутри разворачивается целая инфраструктура: модель, дашборд, бот, ресерч — всё в одном месте, можно вести несколько агентов параллельно.

1
Создай папку проекта

В VS Code: File → Open Folder… → создай новую папку (например, на рабочем столе) и назови по сути задачи: predict-model, commerce-dashboard, rfm-base. Открой её — слева появится дерево файлов. Всё, что Claude будет делать, ляжет сюда.

2
Положи опорники в папку

Создай внутри подпапку материалы-компании/ и закинь туда свой контекст из прошлых домашек: бриф, JTBD, Tone of Voice, VPC, позиционирование, УТП, оффер, продукты + цифры (план/факт, текущая воронка, отчётность). Рядом — инструкции/: плейбуки, шаблоны, чек-листы, транскрипты разборов.

Правило: «контекст, который нельзя не учесть». Слабый вход → слабый результат.
3
(Опционально) Репозиторий и GitHub

Если хочешь историю версий и страховку от поломок — заведи репозиторий. Репозиторий = папка проекта под управлением Git с историей всех изменений; GitHub = облако для репозиториев. Просто скажи Claude:

Инициализируй git-репозиторий в этой папке, сделай первый коммит и помоги выложить на мой GitHub. Объясни шаги простыми словами.

Зачем: любую ошибку можно откатить к рабочей версии. Аккаунт — на github.com.

4
Подключи скиллы и MCP (по необходимости)
  • Скиллы — «надстройки» под роль (маркетолог, аналитик, дизайнер, глубокий ресерч). Делают ответы целевыми. Попроси: найди и подключи скиллы под эту задачу. Готовые — на GitHub.
  • MCP — стандарт подключения внешних сервисов: Telegram, Google, CRM (AmoCRM и др.). Подключается через токен/ключ, который ты генеришь в самом сервисе и скармливаешь Claude.
5
Первая команда
У тебя в папке лежат опорные материалы компании и инструкции. Провались в каждую папку, внимательно изучи, разберись, что это за компания и какие у неё цифры. Дальше с опорой на это мы будем работать.
Готово, когда: Claude пересказал твою компанию и её узкие места своими словами.
03

Две ветки: мультипликатор или продукт

Перед сборкой определись, что именно ты делаешь. Для этого ДЗ почти всем подойдёт ветка А. Ветка Б — если ты решил уйти вглубь и собрать настоящий продукт.

Ветка А · для ДЗ

Внутренний мультипликатор

быстро · усиливает текущий менеджмент

Инструмент, который не идёт на рынок, а ускоряет тебя: модель, дашборд, скрипт, калькулятор, аналитика, бот. Разворачивается за вечер. Это основной формат твоего ДЗ.

  • Опорники + плейбуки → в папку
  • Задача + Definition of Done
  • Ресерч → скиллы → план → сборка
  • Челлендж первой версии → координации
Ветка Б · по желанию

Полноценный продукт

вкус + архитектура · идёт на рынок

Если хочешь собрать сервис/продукт на продажу. Здесь решают вкус (глубина + подача) и архитектура. Стартуй с CLAUDE.md-репозитория, который сначала челленджит твою идею.

  • Челлендж идеи: а точно нужно? кто уже решает?
  • Ресерч рынка: объём, SWOT, PESTEL, Five Forces
  • JTBD + VPC, слой безопасности и базы данных
  • Первые ~60–100 мин — не код, а архитектура
04

Алгоритм сборки · 7 шагов

По этому маршруту собирается любой мультипликатор. Тот же путь, что в живой сборке на эфире — повтори его на своей компании.

1 Контекст
Загрузи опорники в папку

Бриф, JTBD, ToV, VPC, цифры, плейбуки, шаблоны, транскрипты разборов. Чем плотнее вход — тем сильнее результат.

2 Цель + DoD
Опиши образ результата

Definition of Done: количество + качество + ресурсы. «Дашборд воронки, обновляется 2×/сутки, со светофором план/факт».

3 Ресерч
Запусти глубокий research

Бенчмарки по твоим каналам и воронкам, рынок, ЦА. Чтобы увидеть, в каких метриках ты отклоняешься от нормы.

4 Скиллы
Подгрузи целевые скиллы

Под маркетинг, воронки, анализ рынка, позиционирование. Попроси Claude самому найти, чего ему не хватает.

5 План
Пошаговый план + промпт

Попроси действовать как промпт-инженер, задать уточняющие вопросы и самому на них ответить. Роль указывать уже не обязательно — формат ответа всё ещё важен.

6 Сборка
Собери и почелленджь

Первую версию не принимаем. Открой результат, проверь перемножения цифр, дай 1–2 итерации правок.

7 Жизнь
Оживи координацией

Дашборд + бот в Telegram: ежедневная сводка в одно и то же время, алерты по просадкам. Без этого инструмент мёртв.

🎯 Главное

Ценность не в табличке, а в действиях за ней. Назначь ответственного напротив каждой метрики и настрой координации — система должна «капать» напоминанием в одно и то же время.

05

Идеи: выбери своё направление

Меню под разные роли и горлышки. Возьми одно, что бьёт в твою главную метрику. Можно скомбинировать со своими опорниками — это и есть «снайперский выстрел».

📈

Предиктивная модель продаж

для коммерческого директора / РОПа

План/факт по воронке с опережающими и запаздывающими метриками, светофор зон, переключатель «Факт ↔ Норматив» — видно, какой рычаг двигает выручку.

DoD: модель открывается, цифры пересчитываются вживую, подсвечены 2 главных горлышка.
📊

Коммерческий дашборд

для собственника / коммерции

Выручка, маржа, воронка, каналы и UTM в одном экране. Обновление 2–3 раза в сутки, привязка к CRM/аналитике через токен.

DoD: дашборд показывает актуальные цифры по каналам и конверсиям между этапами.
🧮

RFM / LTV-анализ базы

для маркетинга / CRM-маркетолога

Сегментация базы по давности, частоте, деньгам + матрица «что не используют». На выходе — точки cross-sell / up-sell и сценарии реактивации под VPC сегментов.

DoD: база разбита на сегменты, есть список горящих аккаунтов и заходов под их боли.
🔔

Система алертов по касаниям

для РОПа / контроля качества

Регламент касаний → система ловит, если по лиду не выполнен норматив, и шлёт алерт в чат: «по этой сделке менеджер молчит». Логи звонков и сообщений из CRM.

DoD: при нарушении регламента в чат прилетает уведомление с лидом.
💰

Калькулятор мотивации + оффер

для HR / руководителя отдела

Система мотивации нового человека с учётом юнит-экономики продукта и доли дохода. Оффер, разбитый на испытательный срок и после, с подсветкой рисков.

DoD: калькулятор считает оффер по вводным, сходится юнит-экономика.
🎯

Боевой скрипт + онбординг sales

для отдела продаж

Интерактивный скрипт по новой воронке с учётом пресейла и привязкой к продукту. Плюс пошаговый онбординг: что должен знать sales, когда идёт продавать.

DoD: по скрипту реально проходит менеджер, онбординг закрывает первый день новичка.
🏭

Контент-завод

для маркетинга / личного бренда

Из одной папки — стратегия и генерация контента под LinkedIn, Instagram, Telegram, SEO-блог. Ресерч самых охватных тем + черновики постов под стиль каждой площадки, 3×/неделю.

DoD: приходит сводка тем + готовые черновики под площадки.
🧠

«Второй мозг»

для собственника / операционки

Подтягивает источники (Infinity / Notion / календарь / Ганты / рабочий Telegram) и отвечает на голос: статус проекта, где горлышко, какой фокус на неделю. Сам ставит задачи и слоты.

DoD: по голосовому запросу даёт сводку и заносит задачу в трекер.
🧭

VPC + коммерческая стратегия

для маркетинга / стратегии

JTBD и Value Proposition Canvas по core-сегментам → коммерческая стратегия как набор конкретных проектов с DoD на квартал (acquisition + активация).

DoD: по каждому core-сегменту есть VPC и 3–5 проектов с измеримым результатом.
🤖

Бот-модератор по Гантам

для операционного директора

Ежедневно в одно и то же время сканирует проекты/Ганты и пишет сводку: кто что сделал, кто буксует, где нужна помощь. Рука на пульсе вместо ручной модерации.

DoD: в чат приходит ежедневный отчёт по статусам и просрочкам.
06

Безопасность: зелёная и красная зона

Главный риск — не хакер, а ты сам в 3 ночи. Есть реальные истории: одному агент задублировал все контакты в CRM, другому — удалил всю базу. Финальное слово всегда за тобой.

✅ Зелёная зона — можно смело

  • Лимит бюджета в Anthropic Console выставлен заранее.
  • Режим Edit automatically — опасное Claude спрашивает.
  • Работа с файлами, моделями, дашбордами, текстами в папке проекта.
  • Перед записью в боевые системы — бэкап и тест на копии.
  • Доступы к сервисам — через токены/ключи, которые легко отозвать.

⛔ Красная зона — стоп

  • Bypass permissions рядом с боевой базой — не включать.
  • Полный доступ к CRM/БД новичку — нет. Сначала тренируйся на копии.
  • Постоянная смена стран через VPN при логине — риск блокировки.
  • Пароли, ключи, токены — не коммитить в репозиторий.
  • Первую версию выкатывать в прод без проверки — никогда.
Лайфхак из практики: если пускаешь Claude в CRM — прямо в промпте проговори: «убедись, что ничего не удалишь и не задублируешь, перед любой записью покажи мне, что собираешься сделать». Отдельный плейбук по безопасности можно загружать в папку вместе с опорниками — тогда он учитывает риски по умолчанию.

08

Как сдать

Definition of Done твоего ДЗ

Что прислать

1. Скрин/ссылку на работающий инструмент. 2. 2–3 предложения: какое горлышко чинил, что собрал, какой был DoD.

Критерий «зачёт»

Инструмент открывается и работает на твоих цифрах — им можно пользоваться в понедельник, а не просто «получилось собрать».

Бонус (+вкус)

Оживил координацией: дашборд обновляется сам / бот шлёт ежедневную сводку или алерт в Telegram.

Дедлайн

[впишем дату] · вопросы и затыки — в чат потока, помогаем разобраться.

Финальная мысль. Сегодня навык вайб-кодинга — это как умение читать. Мы ограничены только стилем мышления и execution’ом. Возьми одно горлышко, собери инструмент, оживи его координацией — и пусть он работает на тебя каждый день.
