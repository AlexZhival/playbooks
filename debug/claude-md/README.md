# Плейбуки дебага для второго мозга

Набор плейбуков, которые превращают ваш «второй мозг» в дебаг-инженера: он читает вашу предиктивную модель, находит бутылочные горлышки и проводит дебаг вместе с вами — в Claude Code или прямо в Telegram-боте.

## Что внутри
| Файл | Для чего |
|---|---|
| `00-debug-master.md` | **Главный плейбук** — с него начинается любая дебаг-сессия. Оркестрирует остальные. |
| `01-vybor-gorlyshka.md` | Выбор ТОП-1 горлышка: приземление на метрику, PHADI vs HADI, вес в деньгах (ICE). |
| `02-diagramma-ishikava.md` | Декомпозиция горлышка на 6 веток. |
| `03-metod-5-pochemu.md` | Быстрый спуск до корня. |
| `04-derevo-gipotez.md` | Гипотезы + research + провалиться до данных. |
| `05-faktorny-analiz.md` | Вывод в деньгах: топ-3 фактора, «как усилить сильные стороны». |
| `06-sborka-tehnologii-playbook.md` | Action plan на квартал + сборка playbook в библиотеку. |
| `07-debug-ritual-i-telegram.md` | Ритм дебага, каскад по уровням, работа через Telegram-бота. |

## Как установить во второй мозг
1. Скопируйте всю папку `claude-md/` в проект своего второго мозга (рядом с вашим `CLAUDE.md` и папкой `context/`).
2. В `context/` должны лежать материалы про ваш бизнес: воронка, продукт, этап, сегменты (собирали на модуле «Второй мозг»).
3. Подключите предиктивную модель через **Google Sheets API — только на чтение** (роль «просмотр»).
4. В главном `CLAUDE.md` добавьте строку:
   > При запросе дебага читай `claude-md/00-debug-master.md` и веди пользователя по нему.

## Как запускать
- **В Claude Code:** «Прочитай `claude-md/00-debug-master.md` и проведи меня по дебагу по шагам».
- **В Telegram-боте:** напишите боту «дебаг» — он посмотрит модель, найдёт красные зоны и начнёт разбор недели.

## Принцип безопасности
Все плейбуки требуют подключения к модели **только на чтение**. Агент ничего не меняет и не пересчитывает без вашего подтверждения. Токены/ключи — вне кода и вне git.
