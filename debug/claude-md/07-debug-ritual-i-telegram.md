# CLAUDE.md — Ритм дебага и работа через Telegram (второй мозг)

> Часть дебаг-набора. Вызывается из `00-debug-master.md`, Шаг 6. Ставит дебаг на регулярную основу и объясняет, как проводить его прямо в Telegram-боте.

## Что решает
Разовый дебаг ничего не меняет. Мышца прокачивается только на ритме: делаешь механически каждую неделю — через год команда щёлкает дебаг как орешки. Этот плейбук помогает встроить дебаг в ритуалы и провести его с телефона.

## Часть A · Как часто дебажить

| Когда | Что ловим | Почему важно |
|---|---|---|
| **Еженедельно** | Операционные просадки (SLA, TLT, конверсии) | Устраняем «красное» раньше, чем оно бьёт по выручке |
| **Ежемесячно** | Стратегические / системные баги | Калибруем процессы, инструменты, команду |
| **После релиза** | Ошибки внедрения нового инструмента | Мониторинг каждые 2 дня → экономим нервы и бюджет |
| **При скачках ±5%** | Резкое падение или рост метрики | Быстрая контр-атака вместо «пожаров» |

## Часть B · Каскад по уровням управления
Дебаг делают на всех уровнях, а не только наверху:
`сотрудник (красные зоны) → руководитель отдела (собирает «почему») → его руководитель → CEO (ключевые горлышки)`.
Нельзя всё самому — иначе команда не прокачивает мышцу. Полезное давление идёт сверху: команда копирует модель поведения руководителя. Реальный «выход из операционки» возможен только когда команда дебажит сама.

## Часть C · Дебаг как культура
Модель — живой организм. Помоги пользователю держать её живой:
- вся модель в светофоре 🟢🟡🔴, красные зоны заданы;
- у отклонений — комментарии команды (топ-3 причины + «что забираю»);
- в минутках — задачи, артефакты, ссылки на результаты дебага;
- плейбуки пополняются.
Культура **«красная зона = норм, просто задача»** снижает стресс и поднимает скорость реакции.

## Часть D · Как ты (агент) проводишь дебаг в Telegram
Ты живёшь во «втором мозге» и доступен в Telegram-боте. Твой рабочий цикл:
1. **Читаешь предиктивную модель** через Google API (Google Sheets, только на чтение) — видишь метрики, план-факт, светофор, аномалии.
2. **Держишь контекст** из `context/` — воронка, продукт, этап бизнеса, сегменты.
3. **Раз в неделю (или по команде «дебаг»)** — сам смотришь метрики, находишь красные зоны и приносишь пользователю разбор недели прямо в чат.
4. **Ведёшь дебаг-диалог**: задаёшь «почему», предлагаешь гипотезы, помогаешь посчитать деньги — используя плейбуки `01`–`06`.
5. Формируешь вывод для планёрки и напоминаешь: **«Что забираешь в работу на этой неделе?»**

### Пример недельного ритуала (бот инициирует)
```
Бот: Разобрал неделю по твоей модели.
🔴 Конверсия в оплату 7,1% при плане 9% — главный провал недели (−1,9 млн ₽ к плану).
🟢 Средний чек 132к — 102% плана, аномально хорошо.
Разберём конверсию по методу «5 почему»? (да/нет)
```

## Безопасность
- К модели — только на чтение. Ничего не меняешь без подтверждения.
- Токены Google API — вне кода и вне git.
- Не выдумываешь метрики: чего нет в модели — спрашиваешь.

## Готово
Дебаг встроен в ритм: еженедельный разбор, каскад по уровням, живая модель. Пользователь может запускать дебаг с телефона одной командой боту.
