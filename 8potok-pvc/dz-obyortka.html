Домашнее задание — Pro Vibe Coding: обёртка компании в Claude Code

📚 Все материалы потока — страница пакета

Дедлайн: до следующей встречи. Пункты 1-9 — ядро недели. Пункт 10 (Telegram-бот) и «Расширение» — по желанию / для отдельного модуля.
Принцип: сразу забронируй в календаре слоты под домашку. Не заложил время — с вероятностью 99% вспомнишь за час до встречи.
Принцип дня: «5 раз подумай — 1 раз сделай». Сначала контекст и смысл — потом инструмент.

Как это работает (то, что мы прошли на воркшопе): ты собираешь обёртку компании в Claude Code, в готовой папке-проекте. Claude сам читает инструкции из файла CLAUDE.md, ведёт тебя по конвейеру шаг за шагом и сам сохраняет артефакты в папку — тебе не нужно копировать промпты вручную. Артефакты накапливаются в папке = у тебя на глазах рождается «архитектура» контекста компании.

Метод каждого артефакта — Алгоритм AI-домашки (7 шагов):
Контекст → Роль → Промпт → Черновик → Проверил сам (закрыл 🔴) → Финал-аудит → В папку.

ЧЕК-ЛИСТ — 11 ПУНКТОВ

  1. Забронировать в календаре слоты под домашку
  2. Поставить окружение: VS Code + Claude Code (по памятке)
  3. Скачать стартовый набор (.zip) со страницы пакета
  4. Разархивировать zip
  5. Выбрать, где будет жить папка проекта, и положить её туда
  6. Сложить материалы своей компании в папку материалы-компании/
  7. Открыть папку в VS Code → запустить Claude Code → дать первую команду
  8. Пройти конвейер с Claude Code: бриф → эксперт → конкуренты → JTBD/Core → ToV/позиционирование → видение/миссия
  9. Финал-аудит + валидация: закрыть все 🔴, артефакты лежат в папке обёртка/
  10. (по желанию) Настроить Telegram-бота — только осознанно, прочитав памятку
  11. Свериться с безопасностью (3 правила)

Расширение / отдельные модули: раздел 06 «Прогрев и контент-план» — позже; раздел 08 «Онбординг и Завод A-Players» — это отдельный большой модуль, в рамках обёртки его настраивать не нужно.

ПОЯСНЕНИЯ К КАЖДОМУ ПУНКТУ

01 · Календарь + слоты на домашку

Открой календарь и забронируй конкретные слоты под эту работу. В календаре должны стоять чекпоинты на сборку обёртки. Без заложенного времени всё это — просто насмотренность.
Образ результата: в календаре 2-3 повторяющихся слота до следующей встречи.

02 · Поставить окружение (VS Code + Claude Code)

Это инструмент, в котором мы работали на воркшопе. Поставь по памятке Настройка окружения (Claude Code) — там пошагово с нуля для новичка. Как пользоваться интерфейсом — в памятке Как пользоваться VS Code и Claude Code.
- Нужна подписка Claude Pro или Max (без неё Claude Code не запустится).
- ⚠️ Telegram-бота из памятки пока НЕ подключай — это отдельный пункт 10, делается осознанно.
Образ результата: в терминале VS Code запускается команда claude, Claude Code отвечает.

🆘 Не успеваешь поставить окружение? Не блокируйся: временно собери обёртку в обычном Claude Desktop / Claude в браузере — открой файлы конвейера из набора и иди по ним. Но цель недели — выйти именно на Claude Code, там контекст копится в папке системно.

03 · Скачать стартовый набор

На странице пакета нажми «⬇️ Скачать стартовый набор для Claude Code (.zip)». Это готовая папка-проект: внутри CLAUDE.md (мозг-инструкция), конвейер 00-08, и пустые папки для входа и выхода.
Образ результата: файл starter-obyortka-8potok.zip скачан.

04 · Разархивировать zip

Распакуй архив (двойной клик на macOS / «Извлечь всё» на Windows). Получится папка «Обёртка — стартовый набор (8 поток)» со структурой:

CLAUDE.md            ← Claude Code читает её автоматически
НАЧНИ ОТСЮДА.md      ← короткая инструкция запуска
конвейер/            ← инструкции 00-08
материалы-компании/  ← сюда положишь свои исходники
обёртка/             ← сюда Claude сам сохранит артефакты

Образ результата: распакованная папка открывается, файлы на месте.

05 · Выбрать, где будет жить папка проекта

Реши, где она лежит (см. раздел «Где хранить обёртку» в конвейере 00):
- Просто старт — локальная папка на ноуте (например в «Документы»).
- Бэкап + свои устройства — папка внутри десктоп-приложения Google Drive или Яндекс.Диск (не веб!).
- Команда — приватный репозиторий на GitHub (Git), все тянут общий контекст.
Перенеси распакованную папку в выбранное место. Сервер не нужен.
Образ результата: папка лежит там, где ты сможешь её находить и (по желанию) бэкапить.

06 · Сложить материалы компании

Положи в папку материалы-компании/ свои исходники по приоритету: презентации/КП → тексты сайта → прайсы/тарифы → цифры (выручка, средний чек, клиенты, конверсии) → транскрипты встреч и слова основателя. Цифры важнее прилагательных. Чего нет — не страшно, Claude пометит 🔴.
Образ результата: в материалы-компании/ лежат реальные исходники твоей компании.

07 · Запустить сборку в Claude Code

  1. В VS Code: Файл → Открыть папку → выбери папку проекта.
  2. В терминале VS Code набери claude и запусти Claude Code в этой папке.
  3. Напиши Claude Code первую команду:
Прочти CLAUDE.md и начни вести меня по сборке обёртки

Дальше отвечай на его вопросы. Он сам читает конвейер и ведёт тебя по шагам.
Образ результата: Claude Code поприветствовал, прочитал материалы и задал первый вопрос.

08 · Пройти конвейер (ядро домашки)

Claude ведёт тебя по порядку, по одному артефакту за раз. Ты проходишь:
бриф компании (01) → бриф эксперта (02) → конкуренты и отстройка (03) → JTBD и Core-сегменты (04) → Tone of Voice и позиционирование (05) → видение и миссия (07).
На каждом шаге отвечай на уточняющие вопросы своими данными — это шаг 5 алгоритма (валидация). Финальное слово всегда твоё.
Образ результата: в папке обёртка/ появляются файлы артефактов (01-бриф… 07-видение).

09 · Финал-аудит + валидация (критерий «сделано»)

По каждому артефакту Claude делает короткий финал-аудит (противоречия, факты vs догадки). Закрой все 🔴 своими данными. Если 🔴 много — артефакт не готов, добери данные.
Образ результата (критерий «сделано» по всей домашке): обёртка лежит в папке обёртка/ · цифры важнее красивых слов · все 🔴 закрыты · пройден финал-аудит. Это фундамент, который грузим во все следующие домашки.

10 · Telegram-бот — по желанию, осознанно

Хочешь, чтобы Claude писал тебе в Telegram — есть инструкция в памятке Настройка окружения, Шаги 2-4.

⚠️ Внимание: подключение бота даёт доступ к твоему Telegram — это риск (утечка аккаунта). Сначала прочитай памятку целиком, подготовься, и подключай доступ только осознанно. Не обязательно делать это на этой неделе.
Образ результата: (если делал) бот подключён по памятке, токен ты никому не передавал.

11 · Безопасность — 3 правила

Полное — в памятке Безопасный вайбкодинг. На сегодня:
- Ключи и пароли — никогда в git (токен бота, API-ключи).
- Потолок на расходы у ИИ-сервиса.
- ИИ не трогает прод/базу одной командой — только через твоё подтверждение.
Зоны: 🟢 локально/тест — смело, 🔴 прод/деньги/чужие данные — с оглядкой.
Образ результата: понимаешь, где зелёная зона, а где красная.

ОРГАНИЗАЦИОННОЕ

  • Все материалы и промпты — в наборе и на странице пакета. Конвейер 00-08 ведёт за руку; Claude Code читает его сам.
  • На разбор приходим только после валидации. Первую выдачу нейросети без твоей проверки видно за секунду. Обёртка без закрытых 🔴 — это не обёртка, а заготовка.
  • Команда: если работаете вместе — положите папку в приватный GitHub-репозиторий, каждый тянет свежий контекст (см. конвейер 00).
  • Материалы практикума не публикуем и не шерим. Это интеллектуальная собственность, за которую ты заплатил.
  • Вопросы по неделе — в чат (тех.вопросы по окружению — туда же).
