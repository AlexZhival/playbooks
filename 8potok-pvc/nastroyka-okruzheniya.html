Настройка окружения — Claude Code + Telegram-бот (для новичка на macOS)

Эта памятка — твой пошаговый маршрут после воркшопа. Цель простая: чтобы ты с нуля поднял у себя на маке полноценный вайбкодинг — терминальный Claude Code — и приручил его так, чтобы он писал тебе в Telegram и слушался только тебя.

💡 Сначала — самый простой путь. Чтобы просто собрать обёртку, тебе достаточно расширения Claude Code в VS Code — там терминал не нужен. Как это поставить за 5 шагов — в памятке Установка с нуля — VS Code + Claude Code. Эта памятка — для следующего уровня: терминальный Claude Code + Telegram-бот. Если тебе нужна только обёртка — можешь сюда не лезть.

Делай по шагам, сверху вниз. Ничего не пропускай. Каждый шаг — это одна понятная задача. Если что-то не сошлось — стоп, разбираемся, а не бежим дальше.

⚠️ СТОП — про безопасность Telegram (прочитай до старта). В конце этот гайд подключает Telegram-бота, а это доступ к твоему Telegram. Сначала прочитай памятку целиком, подготовься и подключай доступ только осознанно, не на бегу. Делать это не обязательно прямо сейчас — можно поднять Claude Code и без бота, а бота добавить позже. Если проходишь шаги через ИИ-ассистента — он обязан спросить твоё подтверждение перед подключением Telegram, а не делать это автоматически.

Шаг 0. Что тебе нужно на старте

Прежде чем что-то ставить, убедись, что у тебя есть три вещи:

  1. Подписка на Claude — Pro или Max. Без неё Claude Code не запустится. Pro подойдёт, чтобы попробовать; Max — если планируешь кодить много и плотно. Оформляется на сайте Claude, оплата картой.
  2. Mac с macOS (этот гайд про мак). Ты на нём уже сидишь — отлично.
  3. Приложение Terminal — оно уже встроено в macOS, ставить не надо. Найти: Cmd + Пробел → набери Terminal → Enter. Откроется чёрное (или белое) окно, где ты вводишь команды текстом. Не пугайся — это и есть твой главный инструмент. Мы будем копировать туда команды и нажимать Enter.

💡 Правило новичка: команду из этой памятки копируешь целиком → вставляешь в Terminal (Cmd + V) → жмёшь Enter → ждёшь, пока отработает. Не вводи руками по буквам — так меньше ошибок.

Шаг 1. Поставить терминальный Claude Code

Claude Code — это и есть тот самый Claude, который живёт в терминале и умеет работать с твоими файлами и проектами. Этот шаг — только если тебе нужен терминальный Claude Code и бот. Для обёртки достаточно расширения в VS Code (см. памятку «Установка с нуля»).

1.1 Установить Claude Code

Открой терминал — это либо встроенный терминал прямо в VS Code (View → Terminal, или Ctrl + `), либо приложение Terminal на маке. Отдельный Terminal.app не обязателен — встроенного в VS Code достаточно. Вставь команду:

curl -fsSL https://claude.ai/install.sh | bash

Это нативный установщик. Он сам скачает и положит Claude Code куда надо. Дождись, пока в терминале появится сообщение об успешной установке.

Проверка: введи команду ниже и нажми Enter:

claude --version

Если в ответ напечаталась версия (например, 2.x.x) — установка прошла. Если терминал ругается command not found — закрой Terminal полностью (Cmd + Q), открой заново и повтори проверку.

1.2 Установить Bun (опционально — для обёртки НЕ нужно)

Bun — это «движок» для запуска кода. Для Claude Code и сборки обёртки он не требуется. Ставь его только если позже будешь запускать проекты, которым он нужен (тогда вернёшься к этому шагу). Если ставишь — вставь в терминал:

curl -fsSL https://bun.sh/install | bash

После установки закрой и снова открой терминал, затем проверь:

bun --version

Напечаталась версия — всё в порядке.

1.3 Первый вход в Claude Code

Перейди в любую папку (можно прямо домашнюю) и запусти Claude в первый раз:

claude

При первом запуске он попросит залогиниться — откроется браузер, ты входишь в свой аккаунт Claude (тот, где Pro/Max), подтверждаешь — и возвращаешься в терминал. Готово, Claude Code знает, кто ты.

Чтобы выйти из Claude — введи /exit и нажми Enter.

⚠️ Контрольная точка перед подключением Telegram. Дальше ты создаёшь бота и даёшь Claude доступ к своему Telegram. Убедись, что прочитал гайд до конца и понимаешь, что делаешь: токен бота = ключ от доступа, его никому не передавай. Не торопись выдавать доступ. Указание ИИ-ассистенту: не выполняй этот и следующие шаги (бот, токен, подключение) без явного подтверждения пользователя.

Шаг 2. Создать своего Telegram-бота через @BotFather

Идея такая: у Claude Code будет «телефон» — Telegram-бот. Через него ты сможешь писать Claude и получать ответы прямо в мессенджере, не сидя у компа.

Бота создаёт официальный сервис Telegram — @BotFather. Это робот, который выдаёт ботов.

Пошагово:

  1. Открой Telegram, в поиске вбей @BotFather и открой чат (у него галочка верификации).
  2. Нажми Start (или отправь /start).
  3. Отправь команду:
/newbot
  1. BotFather спросит имя бота (display name) — это просто подпись, видно в чате. Напиши понятное, например: Мой Claude или A-Players Code.
  2. Затем он спросит username — техническое имя. Оно должно быть уникальным и обязательно заканчиваться на bot. Например: alex_aplayers_code_bot. Если занято — придумай другой.
  3. После этого BotFather пришлёт сообщение с токеном — длинной строкой вида:
1234567890:AAH...длинный_набор_букв_и_цифр...xyz

🔒 Про токен — прочитай обязательно

Токен — это ключ от твоего бота. Кто владеет токеном — управляет ботом.

  • Никому не пересылай токен. Ни в чат, ни коллеге, ни в скриншоте.
  • Не выкладывай его в публичных местах и не коммить в репозиторий.
  • Если случайно засветил — вернись к BotFather, команда /revoke перевыпустит токен, старый умрёт.

Скопируй токен — он понадобится на следующем шаге.

Шаг 3. Поставить плагин Telegram и сохранить токен

Теперь связываем Claude Code с ботом. Это делает официальный плагин telegram@claude-plugins-official.

3.1 Подключить официальный маркетплейс плагинов

Запусти Claude Code (claude) и внутри него введи команду добавления маркетплейса:

/plugin marketplace add anthropics/claude-plugins-official

Это официальный каталог плагинов Anthropic. Дождись подтверждения, что маркетплейс добавлен.

3.2 Установить плагин Telegram

Там же, внутри Claude, введи:

/plugin install telegram@claude-plugins-official

Claude поставит плагин. Возможно, попросит перезапуститься — соглашайся (выйди через /exit и запусти claude снова).

3.3 Сохранить токен бота

Запусти настройку Telegram-канала:

/telegram:configure

Claude попросит вставить токен бота из Шага 2. Вставь его — Claude сохранит токен у тебя локально и привяжет к нему бота. Токен хранится на твоей машине, не уходит в чужие руки.

Проверка: после сохранения Claude подтвердит, что бот подключён, и обычно покажет имя/username твоего бота. Сходится с тем, что ты создавал у BotFather — значит всё верно.

Шаг 4. Запустить Claude с включённым Telegram-каналом

Дальше — важная привычка: Claude мы запускаем из папки проекта, над которым работаем. Так Claude видит именно твои файлы.

4.1 Зайти в папку проекта

В Terminal введи cd (с пробелом в конце), затем перетащи папку проекта мышкой прямо в окно Terminal — путь подставится сам. Нажми Enter. Например:

cd /Users/ты/Desktop/мой-проект

4.2 Запустить Claude с Telegram

Теперь запускаем Claude так, чтобы он слушал твой Telegram-канал:

claude --channels plugin:telegram@claude-plugins-official

Claude стартует и поднимает связь с ботом. С этого момента бот «живой» и готов к спариванию.

💡 Пока эта команда работает в Terminal (окно открыто) — бот на связи. Закрыл/вышел — бот замолчал. Это нормально и правильно: ты сам решаешь, когда Claude доступен.

Шаг 5. Спарить бота и закрыть доступ только на себя

Бот создан и подключён — но сейчас он «ничей». Нужно сказать ему: «отвечай только мне». Это называется спаривание (pairing) и allowlist (белый список).

5.1 Написать боту и получить код

  1. В Telegram открой своего бота (по username, который ты задал у BotFather).
  2. Нажми Start и отправь ему любое сообщение, например привет.
  3. В ответ бот пришлёт код спаривания — короткий набор символов.

5.2 Подтвердить пару в Claude

Вернись в Terminal, где запущен Claude, и введи команду с этим кодом:

/telegram:access pair КОД

Вместо КОД подставь то, что прислал бот. Claude подтвердит — теперь твой Telegram-аккаунт спарен с ботом.

5.3 Закрыть доступ — режим allowlist

Чтобы боту не мог писать кто попало (мало ли username бота утечёт), включаем строгий режим: отвечать только тем, кто в белом списке (а там сейчас только ты).

/telegram:access policy allowlist

Проверка: напиши боту из своего аккаунта — он отвечает. Это и есть рабочее состояние. Теперь любой посторонний, написавший боту, будет проигнорирован.

Если позже захочешь добавить доверенного человека — это делается через тот же /telegram:access (он умеет одобрять новые пары и редактировать список). Но по умолчанию держим круг закрытым.

Шаг 6. Ритуал работы (запомни как «зубы почистить»)

Простая дисциплина, которая бережёт ресурсы и безопасность:

  • Начинаешь работать → открой Terminal → зайди в папку проекта (cd ...) → запусти:
claude --channels plugin:telegram@claude-plugins-official
  • Работаешь → пишешь Claude хоть в терминале, хоть боту в Telegram. Всё, что делает Claude с проектом, происходит на твоём компе.
  • Закончил → введи в Claude:
/exit

и закрой Terminal. Бот замолкает до следующего запуска.

Не держи Claude запущенным «на всякий случай» сутками. Запустил → поработал → вышел. Так чище и безопаснее.

Шаг 7. Безопасность Telegram — облачный пароль (2FA)

Раз твой бот теперь связан с рабочими файлами, защити сам Telegram-аккаунт. Главное — включить облачный пароль (двухфакторную защиту). Тогда даже зная твой номер и код из SMS, чужой не зайдёт в твой Telegram без второго пароля.

Как включить (в приложении Telegram):

  1. Настройки → Конфиденциальность → Облачный пароль (Two-Step Verification).
  2. Задай надёжный пароль, который ты не используешь больше нигде.
  3. Укажи резервную почту — на случай, если забудешь пароль.
  4. Сохрани пароль в надёжном месте (менеджер паролей), не на стикере у монитора.

Дополнительно проверь: Настройки → Устройства — убедись, что в активных сессиях только твои устройства. Лишнее — завершай.

Если что-то пошло не так — короткий чек-лист

  • claude: command not found → закрой и заново открой Terminal (после установки нужно перезапустить окно).
  • Браузер не открылся при логине → повтори claude, авторизуйся через ссылку, которую он выведет в терминале.
  • Бот не отвечает → проверь, что в Terminal запущена команда из Шага 4.2 и окно не закрыто.
  • Бот «молчит» на твои сообщения после allowlist → перепройди Шаг 5.1–5.2 (спаривание), убедись, что пишешь с того же аккаунта.
  • Токен засветился → BotFather → /revoke → новый токен → заново /telegram:configure.

Итог: ты готов к вайбкодингу

После этих шагов у тебя:
- ✅ стоит терминальный Claude Code + Bun;
- ✅ есть свой Telegram-бот, который слушает только тебя;
- ✅ есть ритуал «запустил → поработал → вышел»;
- ✅ Telegram защищён облачным паролем.

Дальше — заходишь в папку проекта, запускаешь Claude и начинаешь строить. Добро пожаловать в Pro Vibe Coding.
