Раздел 00 · Как пользоваться бриф-конвейером

📖 Глоссарий — термины, которые встретятся

Если видишь незнакомое слово — загляни сюда, не разбирайся «на лету».

Смысл и стратегия:
- Вайбкодинг — получаешь рабочий результат, управляя ИИ словами, а не кодом.
- Обёртка компании — полный смысловой слой о бизнесе (бриф, JTBD, позиционирование…), который грузишь во все задачи.
- Артефакт — один готовый документ обёртки (бриф, JTBD-карта и т.д.).
- Промпт — текстовая команда/запрос к ИИ.
- Системный промпт — первый промпт в чате; задаёт ИИ роль и правила.
- Контекст — все материалы, которые ИИ «видит» и на которые опирается.
- 🔴 пробел — место, где данных не хватило; ИИ метит красным, ты закрываешь своими данными.
- [ГИПОТЕЗА] — вывод ИИ не из факта, а логикой; требует твоего подтверждения.
- Финал-аудит — проверка артефакта на противоречия и «факты vs догадки».
- JTBD (Jobs To Be Done) — «работа», ради которой клиент нанимает продукт.
- Core-сегмент — ядровая группа клиентов (связка задач + триггер), под кого строим.
- ToV (Tone of Voice) — голос бренда: как компания говорит, лексика и запреты.
- Позиционирование — одна фраза: кто ты и почему именно ты.

Инструменты и техника:
- VS Code — бесплатный редактор от Microsoft; «рабочий стол», где открываешь папку проекта.
- Claude Code — Claude, живущий в терминале/VS Code; работает с твоими файлами и папками.
- Claude Desktop — приложение Claude (чат + файлы в одном окне).
- Терминал — окно, куда вводишь команды текстом.
- MD / .md (Markdown) — простой текстовый формат с разметкой; в нём лежат эти инструкции и артефакты.
- CLAUDE.md — особый файл-инструкция: Claude Code читает его автоматически при старте в папке.
- Папка проекта — обычная папка на диске, где живут материалы и артефакты проекта.
- Репозиторий (repo) — папка проекта под управлением Git (с историей версий).
- Git — система версий: хранит историю изменений, позволяет команде работать вместе.
- GitHub — облачный сервис для Git-репозиториев (туда «пушишь» проект).
- MCP (Model Context Protocol) — стандарт, через который ИИ подключается к внешним сервисам (Telegram, Google и т.п.).
- API / API-ключ — способ программам общаться между собой; ключ = пароль доступа, его не светят.
- Токен — то же, что ключ-пароль (например, токен Telegram-бота); хранить в секрете.
- .env / .gitignore — файлы для секретов и для того, что НЕ выкладывать в Git.
- Прод (продакшн) — «боевая» версия (живой сайт, реальные данные). Противоположность — локально (у тебя на ноуте).
- Бот / Telegram-бот — «телефон» для Claude: пишешь в мессенджере — он отвечает.
- noindex / unlisted — страница доступна по ссылке, но скрыта от поиска Google.

👥 Где хранить обёртку и как работать командой

Обёртка — это просто папка с файлами. Выделенный сервер не нужен.
- Один человек: локальная папка на ноуте — Claude Code работает с ней сразу.
- Бэкап + свои устройства: поставь десктоп-приложение Google Drive или Яндекс.Диск (не веб-версию!) — появится синкаемая локальная папка, VS Code её видит.
- Команда над одним контекстом: заведите приватный репозиторий на GitHub (Git). Каждый клонирует его, перед работой делает git pull (берёт свежее), после — git push (отдаёт изменения). Все тянут актуальный контекст, версии сохраняются — это правильнее любого «сервера».

Подробно — в памятке Где хранить папку проекта.

1. Что такое «обёртка» компании и зачем она нужна

«Обёртка» — это полный стратегический слой смыслов о компании, который вы один раз собираете, а потом переиспользуете во всех задачах: тексты сайта, рекламные офферы, скрипты продаж, посты, презентации, онбординг сотрудников. Без обёртки каждый новый текст вы пишете «с нуля» и каждый раз получаете разный, рассогласованный результат. С обёрткой вы кормите ИИ один проверенный документ — и на выходе всё звучит как одна компания.

Бриф-конвейер — это последовательность из 7 артефактов, каждый следующий опирается на предыдущий. Порядок прохождения строго такой:

№ Артефакт Что отвечает На что опирается
1 Бриф Кто мы, что продаём, кому, по чём, какие цифры Исходные материалы клиента
2 JTBD Какую «работу» клиент нанимает нас сделать Бриф
3 Конкуренты / отстройка Кто рядом, чем мы отличаемся, в чём наша уникальность Бриф + JTBD
4 ToV (Tone of Voice) Как мы говорим: голос, лексика, запреты Бриф + отстройка
5 Позиционирование Одна фраза, которая занимает место в голове клиента Всё выше
6 Видение / миссия Куда идём и зачем существуем Позиционирование
7 Онбординг Как новый человек/ИИ за 10 минут понимает компанию Сборка всего выше

Правило конвейера: не перепрыгивайте шаги. Если на шаге 3 всплывает, что в брифе нет цифр по выручке — возвращаетесь и дозаполняете бриф, а не «додумываете». Каждый артефакт — вход для следующего, ошибка в начале умножается к концу.

2. Как кормить ИИ материалами

ИИ силён ровно настолько, насколько богат контекст, который вы ему дали. Перед запуском промптов соберите и загрузите в чат максимум исходников:

Что загружать (по приоритету):
1. Презентации компании / продукта (PDF, питч-деки, коммерческие предложения).
2. Сайт — выгрузите тексты ключевых страниц (главная, о компании, продукты, цены).
3. Описания продуктов / услуг — прайсы, тарифы, состав пакетов.
4. Цифры и оперативная отчётность — выручка, средний чек, кол-во клиентов, доля рынка, юнит-экономика; выгрузка из CRM / отчёт по воронке продаж (конверсии по этапам, Лид→Оплата, CAC по каналам, LTV, план-факт). Это закрывает самые ценные поля брифа. Цифры важнее прилагательных.
5. Транскрипты — записи встреч, интервью с основателем, разбор стратегических сессий, отзывы клиентов дословно.

Как загружать правильно:
- Сначала прикрепите/вставьте все материалы, и только потом дайте промпт. Не пишите «сейчас пришлю» — ИИ начнёт отвечать на пустоту.
- Если материалов много — грузите частями, после каждой пишите: «Принял? Отвечай только "принял", не анализируй». Анализ запускаем отдельным промптом, когда всё загружено.
- Помечайте источник каждого куска: «Это с сайта», «Это транскрипт встречи от [дата]», «Это слова основателя». Это нужно, чтобы на финальном аудите отделить факты от догадок.
- Чего нет — того нет. Не давайте ИИ «придумать» недостающие цифры. Пробелы помечаются 🔴 и закрываются у клиента.

3. Четыре базовых промпта загрузки контекста

Запускаются по порядку: сначала Системный (задаёт правила игры), потом грузите материалы, потом Основной, потом Уточняющий, в конце — Финальный аудит.

3.1. Промпт «Системный» (роль и правила)

Когда запускать: самым первым в новом чате, до загрузки материалов. Задаёт ИИ роль и жёсткие правила, чтобы он не выдумывал.

Ты — корпоративный стратег-аналитик уровня топ-консалтинга. Ты помогаешь
собрать стратегическую «обёртку» компании: бриф, JTBD, отстройку от
конкурентов, Tone of Voice, позиционирование, видение/миссию, онбординг.

ТВОИ ПРАВИЛА РАБОТЫ (соблюдай неукоснительно):

1. ТЫ НЕ ВЫДУМЫВАЕШЬ. Любой факт, цифру или утверждение ты берёшь ТОЛЬКО
   из материалов, которые я тебе загружу. Если данных нет — ты НЕ
   домысливаешь и НЕ подставляешь «типовые» значения.

2. ПРОБЕЛЫ ПОМЕЧАЕШЬ КРАСНЫМ. Везде, где данных не хватает, ставь маркер
   🔴 и формулируй, что именно нужно узнать. Пример:
   «Средний чек: 🔴 нет данных — уточнить у клиента точную сумму в ₽».

3. ОТДЕЛЯЕШЬ ФАКТ ОТ ДОГАДКИ. Если делаешь вывод не напрямую из текста, а
   логически — помечай его как [ГИПОТЕЗА] и проси подтверждения.

4. ССЫЛАЕШЬСЯ НА ИСТОЧНИК. Для важных утверждений указывай, откуда взято
   (сайт / презентация / транскрипт встречи / слова основателя).

5. ПИШЕШЬ КОНКРЕТНО. Цифры, факты, имена — важнее общих прилагательных.
   Никакой «воды», маркетинговых клише и обтекаемых формулировок.

6. НЕ ЗАБЕГАЕШЬ ВПЕРЁД. Делаешь ровно тот артефакт, который я прошу на
   текущем шаге. Не смешиваешь бриф с позиционированием.

Подтверди, что принял правила, и жди — сейчас я загружу материалы о
компании. До команды «анализируй» ничего не анализируй.

Результат / как проверить: ИИ коротко подтверждает правила и ждёт материалы, не начиная анализ. Если он сразу выдал «анализ компании» — правила не приняты, перезапустите промпт.

3.2. Промпт «Основной» (заполнить бриф)

Когда запускать: после того, как ВСЕ материалы загружены в чат и ИИ подтвердил приём.

Все материалы загружены. Теперь проанализируй их и заполни БРИФ компании
по структуре ниже. Бери данные ТОЛЬКО из материалов. Где данных нет —
ставь 🔴 и пиши, что именно надо уточнить. Где вывод логический, а не
прямой — помечай [ГИПОТЕЗА].

СТРУКТУРА БРИФА:

1. КОМПАНИЯ
   — Название, сфера, год основания, география, размер (сотрудники)
   — Чем занимается в одном абзаце «как есть»

2. ПРОДУКТ / УСЛУГА
   — Что именно продаёт (перечень продуктов/услуг)
   — Состав, формат, как устроено потребление
   — Цены / тарифы / средний чек (в ₽, конкретно)

3. КЛИЕНТ (ЦА)
   — Кто покупает: сегменты, портрет(ы)
   — B2B или B2C, лицо принимающее решение
   — Боли и задачи клиента (из материалов, не выдуманные)

4. РЫНОК И ЦИФРЫ
   — Выручка / оборот, динамика
   — Кол-во клиентов, средний чек, повторные продажи
   — Доля рынка / позиция в нише (если есть)
   — Ключевые метрики бизнеса

5. КАНАЛЫ ПРОДАЖ И МАРКЕТИНГА
   — Откуда приходят клиенты сейчас
   — Что работает / не работает (если упомянуто)

6. КОМАНДА И ЭКСПЕРТИЗА
   — Ключевые люди, основатель, в чём сила команды

7. ЦЕЛИ И КОНТЕКСТ ЗАДАЧИ
   — Куда хочет прийти компания (цели, амбиции)
   — Что болит сейчас, зачем делаем эту работу

Формат ответа: по каждому пункту — короткие тезисы с фактами. В конце
дай блок «🔴 ПРОБЕЛЫ» — единым списком всё, чего не хватило для полного
брифа.

Результат / как проверить: заполненный бриф по 7 блокам + отдельный список пробелов 🔴. Проверьте: все цифры действительно есть в материалах (нет «придуманных» сумм), пробелы честно помечены, нет маркетинговой воды.

3.3. Промпт «Уточняющий» (вопросы по пробелам)

Когда запускать: сразу после получения брифа, чтобы превратить пробелы 🔴 в конкретный список вопросов клиенту.

На основе заполненного брифа и всех пробелов 🔴 сформируй СПИСОК
УТОЧНЯЮЩИХ ВОПРОСОВ, которые я задам клиенту, чтобы закрыть все белые
пятна.

Требования к вопросам:
— Группируй по разделам брифа (Продукт, Клиент, Цифры и т.д.).
— Каждый вопрос — конкретный и закрытый по смыслу: чтобы на него можно
  было ответить цифрой, фактом или коротким тезисом, а не эссе.
— Для каждого вопроса в скобках поясни, ЗАЧЕМ он нужен (что мы сможем
  сделать, получив ответ).
— Отсортируй вопросы по важности: сверху — критичные (без них нельзя
  двигаться по конвейеру), снизу — желательные.
— Не задавай вопросы, на которые ответ УЖЕ есть в материалах.

Формат: нумерованный список по группам. В начале — пометка, сколько
вопросов критичных, сколько желательных.

Результат / как проверить: структурированный список вопросов, отсортированный по важности, без дублей с тем, что уже известно. Проверьте: каждый вопрос реально закрывает конкретный 🔴 из брифа, нет «вопросов ради вопросов».

3.4. Промпт «Финальный аудит»

Когда запускать: после того, как клиент ответил на уточняющие вопросы и вы дозагрузили ответы в чат. Финальная проверка качества брифа перед переходом к JTBD.

Клиент ответил на уточняющие вопросы (ответы выше). Обнови бриф с учётом
новых данных и проведи ФИНАЛЬНЫЙ АУДИТ. Выдай результат в таком виде:

1. ПРОТИВОРЕЧИЯ
   — Найди все места, где материалы или ответы противоречат друг другу
     (например, цена на сайте ≠ цена в презентации, разные цифры по
     выручке). Перечисли пары противоречий и попроси решить, какое верно.

2. ФАКТЫ vs ДОГАДКИ
   — Раздели весь бриф на два столбца: что подтверждено материалами
     (ФАКТ + источник) и что осталось [ГИПОТЕЗА] / 🔴.
   — Если гипотез/пробелов всё ещё много — честно скажи, что бриф НЕ
     готов к следующему шагу, и перечисли, что добрать.

3. РЕЗЮМЕ
   — Дай сжатое резюме компании на 5–7 предложений: кто, что, кому, по
     чём, чем силён, куда идёт. Только факты.
   — Поставь вердикт: «Бриф готов к JTBD» или «Бриф не готов, нужно: …».

Не сглаживай. Твоя задача — поймать всё, что может развалить дальнейшую
работу, ПОКА мы в начале конвейера.

Результат / как проверить: список противоречий, чёткое разделение фактов и догадок, сжатое резюме и явный вердикт о готовности. Проверьте: если вердикт «готов», а 🔴 ещё остались — это сигнал вернуться, а не идти дальше.

4. Мастер-промпт для восстановления контекста

ИИ забывает контекст между сессиями. Когда открываете новый чат (или контекст «поплыл» в длинном), не пересказывайте всё руками — вставьте этот промпт и приложите последний готовый артефакт.

Когда запускать: в начале новой сессии/чата, когда нужно продолжить работу над конвейером с того места, где остановились.

Мы продолжаем работу над стратегической «обёрткой» компании. Восстанови
контекст и снова войди в роль.

ТВОЯ РОЛЬ: корпоративный стратег-аналитик. Правила: не выдумываешь;
пробелы помечаешь 🔴; отделяешь ФАКТ от [ГИПОТЕЗА]; ссылаешься на
источник; пишешь конкретно, цифрами, без воды; делаешь ровно текущий
шаг конвейера.

КОНВЕЙЕР (порядок): Бриф → JTBD → Конкуренты/отстройка → Tone of Voice →
Позиционирование → Видение/миссия → Онбординг.

ГДЕ МЫ СЕЙЧАС:
— Компания: [НАЗВАНИЕ КОМПАНИИ]
— Уже готовы артефакты: [например: Бриф, JTBD]
— Сейчас работаем над: [НАЗВАНИЕ ТЕКУЩЕГО АРТЕФАКТА]
— Открытые пробелы 🔴: [перечислить, что ещё не закрыто, или «нет»]

Ниже я прикладываю последний готовый артефакт как опорный контекст:
[ВСТАВИТЬ ТЕКСТ ПОСЛЕДНЕГО ГОТОВОГО АРТЕФАКТА — например, финальный бриф]

Прочитай, подтверди что восстановил контекст одним абзацем-резюме (кто
компания и на каком шаге мы стоим), и жди мою команду на следующий шаг.
Ничего не делай, пока я не дам задачу.

Результат / как проверить: ИИ выдаёт короткое резюме (что за компания + на каком шаге конвейера) и ждёт команду. Проверьте: резюме совпадает с реальностью и ИИ не начал самовольно генерировать следующий артефакт. Если резюме мимо — контекст приложен недостаточный, добавьте артефакты.
